# Improvement Log

This file tracks issues, observations, and improvement ideas from automated
triage runs. Each entry is appended after a run — only when something
noteworthy happened. Clean runs with no issues do not get an entry.

Entries are newest-first. Scan the Category column to find what matters:
**error** entries need attention soonest, then **reliability**, then
**efficiency** and **rule-refinement**.
